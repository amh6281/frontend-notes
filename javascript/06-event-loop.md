# Event Loop

자바스크립트의 비동기 처리 원리와 이벤트 루프의 동작을 정리한다.

---

## 동기와 비동기

- **동기(Synchronous)** : 직렬 방식으로 작업을 처리한다. 하나의 작업이 끝나기 전까지 다음 작업이 시작되지 않는다.
- **비동기(Asynchronous)** : 특정 작업이 끝날 때까지 기다리지 않고 다음 작업을 즉시 시작한다.

```ts
console.log("1");
setTimeout(() => console.log("2"), 0); // 비동기 → 나중에 실행
console.log("3");

// 출력: 1 → 3 → 2
```

자바스크립트는 **싱글 스레드 언어**다. 한 번에 하나의 작업만 처리할 수 있다.

그런데도 여러 작업을 동시에 처리하는 것처럼 보이게 하는 것이 비동기 처리이고, 이를 가능하게 하는 것이 이벤트 루프다.

#### 싱글 스레드인데 어떻게 비동기가 되는가?

자바스크립트 엔진 자체는 싱글 스레드지만, 비동기 작업은 엔진이 직접 처리하지 않는다. 브라우저가 제공하는 **Web APIs**와 **이벤트 루프**가 함께 처리한다.

---

## 이벤트 루프

자바스크립트 런타임 외부에서 비동기 실행을 돕기 위해 만들어진 장치다.

콜 스택이 비어 있고 태스크 큐에 대기 중인 함수가 있으면, 이벤트 루프가 순차적으로 태스크 큐의 함수를 콜 스택으로 옮긴다.

```
┌─────────────┐        ┌──────────────┐
│  Call Stack │◀───────│  Event Loop  │
└─────────────┘        └──────┬───────┘
      ▲                       │ 콜 스택이 비면 큐에서 꺼내옴
      │ 실행                   ▼
┌─────────────┐        ┌───────────────────────────┐
│  Web APIs   │───────▶│ Microtask Q / Macrotask Q │
└─────────────┘  콜백   └───────────────────────────┘
   (타이머·fetch·이벤트)
```

### 콜 스택 (Call Stack)

수행해야 할 코드나 함수를 순차적으로 담아두는 스택이다.

함수를 호출하면 실행 컨텍스트가 생성되어 콜 스택에 푸시되고, 순차적으로 실행된다. 콜 스택은 하나뿐이라서, 최상위 실행 컨텍스트가 종료되어 제거되기 전까지는 다른 태스크가 실행되지 않는다.

### Web APIs

브라우저가 제공하는 기능으로 `setTimeout`, DOM 이벤트, `fetch` 같은 비동기 작업을 처리하는 영역이다.

비동기 작업 자체만 처리하고, 콜백 내부 로직은 실행하지 않는다. 작업이 완료되면 콜백을 큐에 넣고, 실제 실행은 JS 엔진이 콜 스택에서 수행한다.

### 태스크 큐 (Task Queue)

실행해야 할 태스크의 집합이다. 비동기 함수의 콜백이나 이벤트 핸들러가 일시적으로 보관되는 영역이고, 우선순위에 따라 두 종류로 나뉜다.

- **마이크로태스크 큐(Microtask Queue)** : `Promise`의 후속 처리(`then`, `catch`, `finally`), `queueMicrotask`
- **매크로태스크 큐(Macrotask Queue)** : 타이머(`setTimeout`, `setInterval`), I/O(네트워크 요청), DOM 이벤트

마이크로태스크가 매크로태스크보다 우선순위가 높다.

- 이벤트 루프는 콜 스택이 비면 먼저 마이크로태스크 큐의 함수를 가져와 실행한다.
- 마이크로태스크 큐가 모두 비면 매크로태스크 큐의 함수를 하나 가져와 실행한다.

> 매 사이클마다 마이크로태스크 큐를 전부 비운 뒤, 매크로태스크는 하나만 처리한다.

---

## 실행 순서 예제

```ts
console.log("start"); // 1

setTimeout(() => {
  console.log("setTimeout"); // 5 (매크로태스크)
}, 0);

Promise.resolve()
  .then(() => {
    console.log("promise1"); // 3 (마이크로태스크)
  })
  .then(() => {
    console.log("promise2"); // 4 (마이크로태스크)
  });

console.log("end"); // 2
```

1. `start`, `end` : 동기 코드라 콜 스택에서 즉시 실행
2. `promise1`, `promise2` : 마이크로태스크 큐라 먼저 비움
3. `setTimeout` : 매크로태스크 큐라 마지막

> `setTimeout(..., 0)`이라도 매크로태스크라서 마이크로태스크인 `Promise`보다 항상 늦다.

## Promise와 async/await

### Promise가 등장한 이유

콜백 안에 콜백을 중첩하는 콜백 지옥을 해결하고, 비동기 코드를 더 읽기 쉽고 관리하기 쉽게 만들기 위해 등장했다.

- **콜백(Callback)** : 다른 함수에 인자로 전달되어 특정 시점에 나중에 실행되는 함수
- **콜백 지옥** : 실행 순서가 중요한 비동기 작업을 보장하려고 콜백 안에 또 콜백을 중첩하면서 발생

### Promise의 상태

`Pending`(대기)에서 시작해 `Fulfilled`(이행) 또는 `Rejected`(거부)로 바뀐다.

### Promise와 Callback의 차이

Callback은 함수를 전달하는 방식이고, Promise는 비동기 작업의 결과를 객체로 관리해 체이닝과 에러 처리가 쉽다.

### async/await

Promise를 기반으로 비동기 코드를 동기 코드처럼 작성할 수 있게 해주는 문법이다.

```ts
// Promise 체이닝
getData()
  .then((a) => getMore(a))
  .then((b) => console.log(b));

// async/await
const a = await getData();
const b = await getMore(a);
```

#### async/await는 내부적으로 어떻게 동작하나?

`await`는 `Promise.then`의 문법적 설탕(syntactic sugar)이다. `await` 이후의 코드는 마이크로태스크 큐에 등록되어 실행된다.

#### async 함수의 반환값은?

항상 Promise로 래핑된다. `return 1`은 `Promise.resolve(1)`과 같다.

### Promise.all vs Promise.allSettled

- **`Promise.all`** : 하나라도 reject되면 즉시 전체가 reject된다.
- **`Promise.allSettled`** : 모두 완료될 때까지 기다리고, 각 결과(`fulfilled`/`rejected`)를 배열로 반환한다.

> 여러 요청 중 하나가 실패해도 나머지 결과가 필요하다면 `allSettled`를 쓴다.

---

## 요약

- 자바스크립트 엔진은 싱글 스레드지만, 브라우저의 Web APIs와 이벤트 루프 덕분에 비동기 처리가 가능하다.
- 콜 스택은 실행 컨텍스트가 쌓이는 곳이고, 하나뿐이라 최상위 컨텍스트가 끝나기 전까지 다른 태스크는 실행되지 않는다.
- 이벤트 루프는 콜 스택이 비면 큐의 함수를 콜 스택으로 옮긴다.
- 마이크로태스크 큐(Promise)가 매크로태스크 큐(setTimeout 등)보다 우선순위가 높다. 매 사이클마다 마이크로태스크를 전부 비운 뒤 매크로태스크를 하나 처리한다.
- Promise는 콜백 지옥을 해결하려 등장했고, 상태는 Pending에서 Fulfilled 또는 Rejected로 바뀐다.
- async/await는 Promise 기반 문법이며, `await`는 `Promise.then`의 문법적 설탕이다. async 함수의 반환값은 항상 Promise다.
- `Promise.all`은 하나만 실패해도 전체 실패, `Promise.allSettled`는 모든 결과를 배열로 반환한다.
